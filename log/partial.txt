  🐛 Bugs encontrados

  🔴 CRÍTICOS

  Bug #1 — Flags después de positional args NO se parsean

  $ hsme-cli search-exact "ollama" --format=json
  map[results:[{1 1 0 # Technical...]]   ← devuelve TEXT con representación Go de map

  $ hsme-cli search-exact --format=json "ollama"
  { "results": [...] }                    ← JSON correcto

  Causa: flag.NewFlagSet().Parse() stdlib se detiene en el primer non-flag arg. Bug grave de UX y contradice FR-040 ("every subcommand must accept --format"). Cualquier usuario habituado a
  git/kubectl/docker que ponga el flag al final va a ver basura silenciosamente.

  Bug #2 — FormatText es solo fmt.Sprintf("%v", v)

  // cmd/cli/output.go:30
  func FormatText(v interface{}) string {
      return fmt.Sprintf("%v", v)
  }

  Test real:
  $ echo "test" | hsme-cli store --source-type note
  map[memory_id:1009 status:stored]      ← representación bruta de Go map

  Contradice FR-041 ("human-readable output"). El default (text) es horrible para store, search-fuzzy, search-exact, explore, admin/*. Solo status tiene un printer bueno (status.go:137). Los demás
  son inutilizables sin --format=json.

  🟠 ALTO

  Bug #3 — Backup naming inconsistente

  // cmd/cli/admin.go:70
  fmt.Sprintf("engram-%s.db", time.Now().Format("20060102_150405"))

  Produce: engram-20260426_144245.db — local time, underscore separator.
  Spec/data-model dice: engram-<UTC>.db con YYYYMMDDTHHMMSSZ.
  Backups existentes (backup_hot.sh): engram-20260426T041010Z.db (formato correcto).

  Fix trivial: time.Now().UTC().Format("20060102T150405Z").

  Bug #4 — Orden incorrecto en restore: cleanup DESPUÉS del rename

  // cmd/cli/admin/restore.go:41-51
  os.Rename(tmpPath, dbPath)   // ← rename PRIMERO
  ...
  os.Remove(dbPath + "-wal")    // ← cleanup DESPUÉS
  os.Remove(dbPath + "-shm")

  El plan especifica el orden inverso (cleanup → rename) para que no exista una ventana donde la nueva DB conviva con sidecars viejos. SQLite usualmente detecta el mismatch via salt en el WAL
  header, pero no es garantizable.

  🟡 MEDIOS

  Bug #5 — getStatus no cuenta nodos ni aristas del KG

  status.go solo retorna memories + tasks. La spec/data-model exige GraphStats { Nodes, Edges }. El scripts/status.sh:44-45 original las contaba. Regresión funcional vs el script reemplazado.

  Bug #6 — --interval flag no implementado

  status.go:49 tiene time.Sleep(2 * time.Second) hardcoded. Spec FR-022 explícitamente exige --interval <duration>.

  Bug #7 — --watch sin verificación TTY

  Spec/research dice que --watch debe fallar si stdout no es TTY. Implementación silenciosamente sigue refrescando sin clear screen, llenaría un archivo si se redirige.

  🟢 BAJOS

  Bug #8 — tmpPath hardcoded sin O_EXCL

  restore.go:34: tmpPath := filepath.Join(dbDir, "restore.db.tmp") — colisiona en restores paralelos, os.Create sobreescribe.

  Bug #9 — runStore mezcla WriteError con fmt.Println plain

  Inconsistencia: algunos errores respetan --format=json, otros son plain text.

  Bug #10 — Faltan flags top-level --db, --ollama-host, etc.

  El help top-level los lista, pero flag.Parse() en main.go no se entera del --db global porque los flags están registrados con RegisterDBFlags(flag.CommandLine, &cfg) y luego flag.Parse() se
  detiene en el subcomando. Para que hsme-cli --db /tmp/x.db status funcione necesitarías parsear globales explícitamente. Lo evidencia tests/modules/cli_test.go:34: append([]string{"--db", dbPath},
   args...) — se ve que pasa el --db ANTES del subcomando, y eso sí funciona, pero confirma que después no.
